# Product

## Register

product

## Users

CodeGo serves developers and technical teams who switch between multiple AI coding tools and provider backends across daily local workflows. They work in desktop environments, often manage several CLI tools at once, and need a reliable way to keep provider, token, session, prompt, MCP, and agent configuration aligned without editing raw config files by hand.

## Product Purpose

CodeGo is a desktop control center for Claude Code, Claude Desktop, Codex, Gemini CLI, OpenCode, OpenClaw, and Hermes. It replaces scattered config-file editing with one operational surface for switching providers, inspecting auth state, managing tool configuration, and handling supporting workflows like sessions, prompts, MCP, skills, diagnostics, and quota visibility. Success means the user can move between tools and providers quickly, understand current state immediately, and trust that local configuration changes are applied safely.

## Brand Personality

Pragmatic, clear, dependable. The product should feel like a serious developer tool: quiet, efficient, and competent rather than flashy. It should reduce friction, surface important state early, and make complex multi-tool setup feel controlled.

## Anti-references

- Marketing-style desktop shells with oversized hero sections, decorative gradients, and weak operational hierarchy
- Over-designed SaaS dashboards that bury primary actions under card-heavy decoration
- Inconsistent mixed-language UI where core navigation, auth state, and setup flows read like different products
- Novel navigation patterns that make standard tool-management tasks harder to scan and repeat

## Design Principles

- Put operational clarity first: current tool, current state, and next action should be obvious at a glance
- Keep one coherent shell: dashboard, provider management, and auxiliary workspaces should feel like one product
- Reduce visual noise: color and emphasis are reserved for active state, warnings, and primary actions
- Design for repeated use: navigation, spacing, and controls should support frequent switching and configuration work
- Respect local trust boundaries: auth, tokens, diagnostics, and secure-storage state must read as careful and explicit

## Accessibility & Inclusion

Target WCAG 2.1 AA contrast and focus visibility across light and dark themes. Keyboard navigation should remain usable for primary navigation and dialogs. Motion should stay restrained and stateful, with no essential information hidden behind animation. Chinese should remain the default first-run language, while English stays fully available.
