## shu26.cfd 是什么？

**shu26.cfd** 是一个面向 AI 编程工作流的大模型中转与模型接入平台，重点服务以下场景：

- Codex 中转
- Claude Code 中转
- Gemini CLI 中转
- OpenAI 兼容接口接入
- AI 编程工具与 Agent 工具接入

它更适合重视稳定性、模型选择、账户统一管理和桌面端配套体验的开发者与团队。

## CodeGo 是什么？

**CodeGo** 是 shu26.cfd 的桌面控制中心。  
你不需要再为不同工具反复手动修改 JSON、TOML、环境变量或散落在本地的配置文件，而是可以通过一个桌面应用完成登录、API 配置、密钥选择、模型查看、日志检查和网站信息同步。

CodeGo 适合这些需求：

- 想把 Codex、Claude Code、Gemini CLI 的中转配置集中管理
- 想减少手动改配置文件的频率
- 想在桌面端直接查看密钥、模型、日志和账户状态
- 想让网站账户和本地工具配置更顺畅地联动

## 为什么推荐 CodeGo

- **围绕 AI 编程中转场景设计**：核心就是服务 Codex 中转、Claude Code 中转、Gemini CLI 中转等桌面使用链路。
- **桌面端统一管理**：把原本分散在多个工具和多个配置文件中的设置收敛到一个桌面应用中。
- **网站和软件配合使用**：shu26.cfd 负责账户、密钥、模型与平台页面，CodeGo 负责桌面端接入和本地管理。
- **跨平台可用**：支持 Windows、macOS、Linux。

## 典型使用场景

### Codex 中转桌面方案

如果你希望使用 **Codex 中转站**，可以在 shu26.cfd 上完成账户相关操作，再通过 CodeGo 在本地完成接入、密钥选择、模型映射和桌面端配置。

### Claude Code 中转桌面方案

如果你需要 **Claude Code 中转**，CodeGo 可以减少手动切换配置的成本，让你更方便地把网站账户能力应用到本地工具。

### Gemini CLI 中转桌面方案

如果你使用 **Gemini CLI 中转**，CodeGo 可以帮助你把本地配置和网站侧账户信息放到一个统一流程中处理。

## CodeGo 的主要用途

- 连接 shu26.cfd 账户
- 管理桌面端 API 配置
- 选择并应用可用密钥
- 获取模型列表并进行本地配置
- 在桌面端查看模型广场
- 在桌面端查看分组状态
- 查看令牌和请求日志
- 通过官方发布通道接收桌面端更新
